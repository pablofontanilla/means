# CLAUDE.md

Guidance for Claude Code when working in this repo.

## What This Is

This directory is **both** the `means` application repo **and** its own
single-repo workspace, managed by the `workspace` Claude Code plugin. There
are no external repos and no bundled domain — the workspace wraps this one
checkout so the workspace skills can organize task work in place.

- **App source of truth:** the code here (`src/`, `tests/`, etc.).
- **Design & planning docs:** see [Docs](#docs) below.
- **Workspace config:** `dev-env.yaml` (single-repo, `repos: []`, lightweight
  mode — no worktrees).

## Task Workflow

Per-task working context lives under `projects/<task-name>/`. Use the
workspace skills to manage it:

| Skill | Description |
|-------|-------------|
| `/workspace:new` | Scaffold a new task workspace under `projects/` (context, checklist, notes). Lightweight mode: no repo selection, no worktree — you edit the source directly. |
| `/workspace:resume` | Reload an existing project's context and continue. |
| `/workspace:consolidate` | Archive completed checklist items from a bloated project `CLAUDE.md`. |

Launch `claude` from this directory so the skills resolve this folder as the
workspace root (it contains `dev-env.yaml`).

## Isolation Model

Lightweight — no git worktrees. `/workspace:new` creates a context/tracking
project under `projects/`; you make code changes directly in this checkout on
whatever branch you choose. To switch to per-task worktree isolation later,
add a real entry to `dev-env.yaml`'s `repos:` list.
